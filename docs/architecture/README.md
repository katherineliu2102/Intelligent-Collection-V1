# 架构文档（architecture/）

> 给**新进场后端同学**的设计落点。重大设计决策以 ADR（Architecture Decision Record）形式沉淀于此，避免散落在聊天记录里。

## 放什么
- 模块/分层的设计取舍、为什么这么拆
- 引入新依赖、新中间件（如 Redis Stream、XXL-Job）的决策记录
- 跨模块影响较大的方案

## 不放什么
- 已有的核心规格（架构设计/核心引擎/领域模型/基础设施）仍在 [`../`](../) 根级，见 [`../README.md`](../README.md) 总索引
- 渠道相关设计见 [`../channel/`](../channel/)（编排同事维护）

## ADR 命名约定
`NNNN-短标题.md`，例如 `0001-引入redis-stream事件总线.md`。建议结构：背景 / 决策 / 备选项 / 影响。
