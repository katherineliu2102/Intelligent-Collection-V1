## 一、单条短信发送（黑名单等拦截错误实时返回）

*   **接口地址**：`http://43.142.92.38:9511/api/send-sms-single\`
    
*   **请求头Content-Type**：`application/x-www-form-urlencoded`
    
*   **请求方式**：`POST`
    
*   **返回值**：`JSON`
    

### 请求参数

| 参数名 | 参数介绍 | 示例 | 必填 |
| --- | --- | --- | --- |
| `sp_id` | 产品sp\_id | `585899` | 是 |
| `mobile` | 手机号 | `176xxxxxxxxx`(支持AES) | 是 |
| `content` | 短信内容 | `【测试】验证码123`(支持AES) | 是 |
| `signature` | 签名（和密码二选一即可，参考签名算法） | `a92569901189f2574effc957c096c432e2d47635` | 否 |
| `password` | 密码（和签名二选一），通过sp\_id的密码进行md5处理 | `bcfd86306642cf75262b0bc515b73bda` | 否 |
| `ext` | 扩展号 | `123` | 否 |

### 返回值

| 参数名 | 参数介绍 | 示例 |
| --- | --- | --- |
| `code` | 详情见错误表 | `0` |
| `msg` | 成功-success 错误-其他提示 | `success` |
| `msg_id` | 消息id，用来获取状态报告时候匹配消息 | `17` |
| `data` | 失败有返回，详情见错误表 | `WL:NMLJ` |

## 五、获取状态报告(每条短信状态仅可获取一次)

*   **接口地址**：`http://43.142.92.38:9511/api/report\`
    
*   **请求方式**：`GET`
    
*   **返回值**：`JSON`
    
*   **说明**：运营商返回某条短信状态后，此接口就能拉取到，之后调用此接口将不会返回此条短信的状态，所以调用者拉到数据后需要根据 `手机号+msg_id` 将状态存在本地。
    

### 请求参数

| 参数名 | 参数介绍 | 示例 | 必填 |
| --- | --- | --- | --- |
| `sp_id` | 产品sp\_id | `585899` | 是 |
| `signature` | 签名（和密码二选一即可） | `a92569901189f...` | 否 |
| `password` | 密码（和签名二选一即可） | `bcfd86306642c...` | 否 |

### 返回值

| 参数名 | 参数介绍 | 示例 |
| --- | --- | --- |
| `code` | 详情见错误表 | `0` |
| `msg` | 成功-success 错误-其他提示 | `success` |
| `data` | 扩展号, msg\_id, 手机号, 状态, 时间, 售价 字段以`,`分隔 多条记录以\` | \`分隔 |

## 六、推送状态报告(每条短信仅推送一次)

*   **注意事项**：推送使用的是`POST`方式，utf8原始字节流。
    
*   **推送格式**：扩展号, 调用发送接口返回的msg\_id, 手机号, 状态, 时间, 售价 字段以`,`分隔 多条记录以`|`分隔。
    
*   **内容示范**：`123,2,1762532xxxx,DELIVRD,2021-12-23 01:02:03,0.2|123,2,17862532xxxx,DELIVRD,2021-12-23 01:02:03,0.1`

## 七、获取用户上行回复(比如退订等，每条短信状态仅可获取一次)

*   **接口地址**：`http://43.142.92.38:9511/api/get-reply\`
    
*   **请求方式**：`GET`
    
*   **返回值**：`JSON`
    
*   **说明**：手机回复上行后，此接口就能拉取到上行数据，每个上行数据只能拉取到一次，所以调用者拉到上行后，需要将上行的内容存在本地，然后自行根据 `msg_id`，`手机号` 等信息匹配具体哪条短信产生的上行回复。
    

### 请求参数

| 参数名 | 参数介绍 | 示例 | 必填 |
| --- | --- | --- | --- |
| `sp_id` | 产品sp\_id | `585899` | 是 |
| `signature` | 签名（和密码二选一即可） | `a92569901189f...` | 否 |
| `password` | 密码（和签名二选一即可） | `bcfd86306642c...` | 否 |

### 返回值

| 参数名 | 参数介绍 | 示例 |
| --- | --- | --- |
| `code` | 详情见错误表 | `0` |
| `msg` | 成功-success 错误-其他提示 | `success` |
| `data` | urlencode后的内容, 手机号, msg\_id, 扩展号, 时间 字段以`,`分隔多条记录以\` | \`分隔 |

## 八、推送用户上行回复(每条上行回复仅推送一次)

*   **注意事项**：推送的是`POST`方式，utf8原始字节流。
    
*   **推送格式**：urlencode后的内容, 手机号, 扩展号, 时间 字段以`,`分隔多条记录以`|`分隔。
    
*   **内容示范**：`TD,158114xxxx,048842426601,20220325123304`
